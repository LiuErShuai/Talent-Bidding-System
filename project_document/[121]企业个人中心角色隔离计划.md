# 背景与目标
当前个人中心统一指向学生视图，企业用户无法看到企业资料。目标：基于路由级隔离，为学生保留现有个人中心，新增企业个人中心页面（展示企业资料、对接信息、资质与项目数据），并更新导航/守卫指向。

# 目录
1. 现状梳理  
2. 方案选择  
3. 实施计划（todolist）  
4. 交付与验收

# 1. 现状梳理
- 路由：`/user` 指向 `src/views/user/Profile/index.vue`，无角色限制。  
- 守卫：`meta.role` 可做校验，`/user` 未设置，企业访问仍落入学生页面。  
- 导航：头像下拉与“个人中心”按钮统一跳转 `/user`。  
- 数据：`userStore.fetchUserProfile` 提供个人资料，企业字段需兼容占位或后续对接。

# 2. 方案选择
- 采用路由级隔离：  
  - `/user` 仅学生可访问（`meta.role: 'student'`）。  
  - 新增 `/enterprise/profile`（或同义）挂载企业个人中心组件，`meta.role: 'enterprise'`。  
  - 导航与头像快捷菜单按角色跳转对应路径。  
  - 守卫角色不符时，企业用户跳转 `/enterprise/profile`。

# 3. 实施计划（todolist）
- 路由调整：`/user` 加角色限制；`/enterprise/profile` 新增并接入 `meta.role: 'enterprise'`。  
- 组件开发：创建 `src/views/enterprise/Profile/index.vue`，包含企业概览、基础资料、联系人、资质/文件、项目统计模块，支持从 `authStore/userStore` 读取数据并提供占位。  
- 导航对齐：`AppHeader` 的“个人中心”入口与 `quickActions` 企业分支指向 `/enterprise/profile`，兜底跳转逻辑按角色返回正确页面。  
- 守卫兜底：`router/guards.js` 角色不符时企业重定向 `/enterprise/profile`。  
- 文档同步：更新 `[000]功能模块文档.md` 增加本次变更记录。

# 4. 交付与验收
- 访问 `/user`：学生正常，其他角色被拦截/重定向。  
- 访问 `/enterprise/profile`：仅企业可见，展示企业信息模块，无报错。  
- 导航头像与“个人中心”入口在企业身份下进入企业页面；学生保持原路径。  
- 文档记录已更新，代码中文注释，未新增测试文件。
