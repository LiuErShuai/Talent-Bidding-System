# 背景与目标
- 背景：导航栏需要统一为“首页/项目大厅/我的项目”，并在头像右侧增加“个人中心”“消息”文本链接（字体细、无边框装饰）。相关入口需登录守卫，未登录时弹出登录对话框。
- 目标：调整导航结构与样式，新增右侧链接，完善路由守卫，保持现有布局简洁不拥挤。

# 1. 需求与范围
- 顶部导航统一三项：首页 `/home`、项目大厅 `/projects`、我的项目（按角色跳转）。
- 头像右侧新增文本链接：个人中心 `/user/profile`、消息 `/messages`，字体正常（400）、无边框/背景/下划线。
- 登录守卫：我的项目、个人中心、消息需登录；未登录点击时提示并弹出登录框，登录后按原意图跳转。
- 角色路由映射（我的项目）：
  - 学生：`/my-projects`
  - 企业：`/enterprise/my-projects`
  - 教师：`/teacher/my-projects`
  - 管理员：`/admin/pre-review`（暂作“我的项目”入口）

# 2. 实施方案
- 模板调整：移除按角色分支的导航，改为统一三项；“我的项目”使用点击触发登录守卫后再跳转。
- 右侧链接：在头像组件右侧新增两个 span/router-link，类名如 `header-link`，控制字体与 hover 色。
- 守卫逻辑：复用 `handleRequireLogin` 或新增 `requireLoginAndGo(path, feature)`，对三处入口校验登录并处理 pendingRedirect。
- 样式：`header-link` 字重 400、14px，颜色与未激活导航一致，hover 变主题蓝；无边框/背景。保持与头像间 12-16px 间距。
