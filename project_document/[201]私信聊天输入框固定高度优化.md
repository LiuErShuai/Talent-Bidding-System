# 背景与目标

- **背景**：当前私信聊天输入框使用 `autosize` 自动增高，输入较多内容时会挤压聊天记录区域，影响 QQ/微信式的固定聊天窗口体验。
- **目标**：将输入框改为固定高度，内容超出时在输入框内部滚动查看/编辑（滚动条随内容出现）。

## 目录

1. 需求与验收标准
2. 实施方案
3. Todolist（时间戳原则）
4. 风险与回滚

## 1. 需求与验收标准

- 私信聊天输入框高度固定，不随内容自动增高。
- 当输入内容超出可视区域时，可在输入框内部滚动查看与编辑。
- 发送快捷键行为不变：`Enter` 发送，`Shift+Enter` 换行。

## 2. 实施方案

- 移除 `el-input` 的 `autosize` 配置，使用固定 `rows` + `CSS` 强制高度固定。
- 对 `.el-textarea__inner` 设置：
  - 固定 `height/min-height/max-height`
  - `overflow-y: auto`（内部滚动）
  - `resize: none`（防止用户拖拽改变尺寸）

- 风险：固定高度在小屏幕下可能显得偏高。
  - 处理：后续可在移动端媒体查询中降低固定高度（例如 72px）。
