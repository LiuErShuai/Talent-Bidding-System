## 一、项目概述

### 1.1 项目背景
基于红山开源创客平台的"揭榜挂帅"模式，打造一个面向学校、企业、学生三方的产教融合平台，实现企业真实项目需求与学生技能培养的有效对接，促进校企合作、学生实践能力提升。

### 1.2 项目研究内容及目标
构建产教融合项目揭榜挂帅、创客任务发布子系统，聚焦“项目对接低效”、“学生参与动力不足”等问题，打造“智能撮合+动态竞榜”双引擎平台，实现企业需求与学生能力的精准匹配，构建 “揭榜-挂帅-考核-转化”全链路闭环。主要解决以下几个问题：
一、需求解析智能化，打破企业项目书与学生简历的语义壁垒，利用大模型自动提取项目技术指标（如 “需要掌握Python爬虫框架”）与学生技能标签（如“具备Scrapy实战经验”），实现机器可读的结构化匹配；
二、参与形式游戏化，引入 “揭榜积分”、“挂帅段位”、“创客勋章”等机制，将项目参与转化为可积累的数字资产，解决传统实践项目 “参与一次即结束” 的粘性不足问题；
三、过程管理透明化，通过对方案提交、中期答辩、成果验收等进行过程跟踪及公式，确保校企双方权益可追溯，解决成果归属争议等管理难题。


- **对接学校-企业**：企业发布真实项目需求，学校组织学生承接
- **对接学校-学生**：学生个人或团队揭榜项目，提升实践能力
- **产教融合**：实现教育链、人才链与产业链、创新链的有机衔接
- **能力认证**：通过项目完成情况评估学生专业能力

#### 建设目标：
产教融合项目揭榜挂帅、创客任务发布子系统主要建设任务包括：
（1）智能撮合引擎，通过语义解析中台，基于大模型提取企业项目书技术指标（支持 PDF/Word 非结构化文本），构建包含 100 + 技术标签的能力图谱；
（3）双向匹配算法，学生端支持“一键智能匹配”，企业端可按 “专业排名前 30%”“竞赛获奖经历” 等条件精准筛选，匹配耗时＜2 分钟；
（3）动态竞榜平台，游戏化成长体系：设置 “青铜 - 王者” 5 级挂帅段位，依据项目完成质量动态升降，解锁专属权益（如王者段位可申请企业导师 1v1 指导）


### 1.3 参考平台分析（红山开源创客平台）

#### 核心数据统计
- 创客任务总数：778个
- 创客人数：954人
- 当前奖金池总额：9510.4万元
- 已支付总金额：4978.3万元
- 任务领域：23+个

#### 核心功能模块
1. **任务管理系统**
   - 任务发布、编号管理
   - 任务状态流转（6个阶段）
   - 任务分类（23+领域）
   
2. **用户角色系统**
   - 发布方、创客、专家三种角色
   
3. **任务生命周期**
   - 成果征集中 → 成果评选中 → 公示中 → 协议签订中 → 支付中 → 已完成/已关闭

| 字段类型 | 字段列表 |
|---------|---------|
| **必填字段** | 项目名称、项目编号（自动生成）、项目领域分类、项目背景描述、项目目标与要求、成果形式、功能指标及要求、性能指标、质量标准、项目周期、奖金额度/实习机会、揭榜模式（单人/团队）、需求人数、联系人信息 |

查看待审核项目列表、审核项目内容合规性、审核项目可行性、审核奖金合理性、通过/驳回操作、填写审核意见、设置项目推荐度

| **搜索功能** | 项目编号搜索、项目名称关键词搜索、企业名称搜索、全文搜索 |

#### 3.2.4 项目详情页

| 区域 | 内容 |
|------|------|
| **基本信息区** | 项目状态标签、项目名称、发布企业、项目领域、奖金额度、揭榜模式、项目编号、发布时间、截止时间、剩余时间倒计时、已揭榜人数、关注人数 |
| **详细描述区** | 项目背景、项目目标、成果形式、功能要求、性能指标、质量标准、评审方式、知识产权归属、联系方式 |
| **附件下载区** | 需求文档下载、参考资料下载 |
| **项目进度时间轴** | 项目发布 → 揭榜提交（剩余X天）→ 成果评审（X天）→ 结果公示（X天）→ 协议签订（X天）→ 奖金发放（X天）→ 项目完成 |
| **操作按钮区** | 我要揭榜、关注项目、分享项目、咨询企业 |
| **标签页切换** | 项目详情、揭榜名单、成果展示 |
- 评论区

### 3.3 揭榜与团队管理模块

### 3.4 成果提交与管理模块

> **对应原型设计页面**：
> - 原型设计文档 [3.8 成果提交页面](#38-成果提交页面)（成果提交）

#### 3.4.1 成果提交（学生端）

| 项目 | 内容 |
|------|------|
| **提交信息** | 成果标题、成果描述、成果文档上传（文档、代码、视频等）、成果演示链接、提交时间记录 |

|------|------|

#### 3.4.4 成果排名与公示
根据评分自动排名、确定中标名单、公示期设置（默认5天）、公示内容（中标人/团队、成果简介、评分）、申诉机制

#### 3.11.1 项目管理工作流

| 阶段 | 流程内容 |
|------|---------|
| **阶段0：立项预审** | 企业草稿项目 → 完成需求模板与预算校验 → 知识产权/保密协议确认 → 学校立项初审 |（原型设计文档：3.14）
| **阶段1：项目发布** | 企业正式提交 → 学校终审 → 审核通过后发布到榜单 |（原型设计文档：3.15）
| **阶段2：揭榜征集** | 学生浏览项目 → 个人/团队提交揭榜申请 → 企业审核揭榜（可选）→ 揭榜成功 |（原型设计文档：3.3、3.5）
| **阶段3：方案提交** | 揭榜成功后提交实施方案 → 多角色联合审核 → 通过/限期修改/驳回 |（原型设计文档：3.5）
| **阶段4：项目执行** | 学生执行项目 → 过程跟踪与监控 → 里程碑节点检测、问题预警 |（原型设计文档：3.6、3.10）
| **阶段5：中期答辩** | 提交中期材料 → 专家/教师评审 → 通过/需改进/不通过 |（原型设计文档：3.16）
| **阶段6：成果提交** | 上传最终成果与文档 → 学校初审 → 通过/要求修改/驳回 |（原型设计文档：3.8）
| **阶段7：成果评审** | 企业+专家复审 → 多维评分与排名 → 确定中标名单 |（原型设计文档：3.9）
| **阶段8：公示与收尾** | 公示结果 → 申诉处理 → 协议签订 → 支付与归档 |（原型设计文档：3.17、3.18）

**流程可视化**：以BPMN/状态机驱动的项目进度时间轴，当前阶段高亮显示、已完成阶段打勾、未完成阶段灰显，所有关键时间节点均带倒计时与负责人提示。

#### 3.11.2 阶段0：立项预审

| 项目 | 内容 |
|------|------|
| **触发条件** | 企业拟定新项目并创建草稿，填写标准化需求模板（背景、目标、交付件、预算、知识产权归属、保密要求）。 |
| **关键动作** | 系统自动校验字段完整度、预算区间、禁止词；企业上传附件（需求书、附件、参考资料）并签署线上保密与知识产权声明。 |
| **审核机制** | 学校管理员进行立项初审，重点校验政策合规性、团队适配度、是否涉及敏感方向。支持“修改后再审”状态，减少重复提交流程。 |
| **产出** | 审核通过后转入“待发布”池，生成项目唯一编号与阶段0审计日志；未通过时返回修改意见并保留草稿版本。 |

| 项目 | 内容 |
|------|------|

| **必备内容** | 项目理解与分解、技术方案、实施计划（比如甘特图/WBS等）、团队分工、预期成果、风险评估及附件，系统支持模版校验与附件批量上传。 |
| **审核机制** | 学校管理员/指导教师/企业可并行或串行审核，评分维度：可行性30%、技术30%、计划20%、分工20%。支持“条件通过”与“驳回”两类结果，并自动生成整改清单。 |
| **版本与权限** | 方案支持版本控制、历史对比及回滚，访问范围可配置（学生、企业、学校、导师）；通过后自动进入阶段4并解锁里程碑计划。 |

| 项目 | 内容 |
|------|------|
| **进度可视化** | 进度条显示0-100%，颜色分级（绿/黄/红），里程碑旗标（方案、中期、成果、自定义），并提供横向时间轴与甘特同步。 |
| **过程记录** | 自动归档方案版本、中期答辩记录、成果提交记录、沟通纪要、变更日志，生成不可篡改的审计链路。 |
| **过程监控** | 学校管理员查看整体项目池、异常/逾期列表并导出报表；教师聚焦所带团队，企业查看自己项目。逾期提醒分为提前3天、当天、超期每日推送。 |
| **进度报告** | 学生提交周报/月报（本周完成、下周计划、问题与风险），系统可生成健康度（红/黄/绿）并推送导师点评；必要时创建问题/风险项并指定责任人。 |
| **辅助能力** | 支持AI助手对周报、风险进行摘要，为管理者提供“问题热度”与“资源需求”建议；必要时触发临时评审或专家辅导。 |

#### 3.11.7 阶段5：中期答辩

| 项目 | 内容 |
|------|------|
| **答辩准备** | 项目周期过半自动锁定时间，提前7天推送流程与资料清单，并要求上传演练视频或预审材料。 |
| **答辩内容** | 进度对比、功能演示、问题与解决方案、后续计划调整，若无法现场参加可提交PPT+录制视频作为替代。 |
| **评审流程** | 3-5名专家/教师评分，维度（进度40%、技术30%、问题解决20%、表达10%），结果分为通过、需改进、不通过，并关联积分奖惩。 |
| **记录留痕** | 答辩时间地点、评委、评分表、意见、录制视频、PPT均入库，作为后续风险判断与最终评审的重要依据。 |

#### 3.11.8 阶段6：成果提交

| 项目 | 内容 |
|------|------|
| **提交要求** | 学生上传最终代码包/部署地址、产品说明书、测试报告、用户手册、演示视频等，系统校验格式、病毒与完整性。 |
| **学校初审** | 重点检查合规性、内容完整性、开源/密级要求，必要时要求补交或整改，支持多轮版本管理。 |
| **关联动作** | 提交成功自动生成验收清单、对照方案与中期结果，异常项会提示在阶段7重点关注；同时更新进度100%待评审状态。 |

#### 3.11.9 阶段7：成果评审

| 项目 | 内容 |
|------|------|
| **评审组织** | 企业负责人+外部专家+校内导师共同组成评审团，可配置线上/线下评测及自动化测试结果接入。 |
| **评分机制** | 维度包括功能完整性、技术质量、创新性、实用性、交付规范、合作反馈等，并支持加权与去极值；系统自动生成排名与建议中标名单。 |
| **评审输出** | 形成电子评分表、专家意见、复议通道及风险标签，若出现争议可触发再次评审或加邀第三方。 |
| **数据沉淀** | 评审数据将回流到学生画像与企业满意度指标，为智能撮合与激励体系提供依据。 |

#### 3.11.10 阶段8：公示、申诉与收尾

| 项目 | 内容 |
|------|------|
| **公示内容** | 中标团队信息、成果简介、各维度分数、专家意见、项目段位加成、知识产权归属说明，公示期默认7天（可配置3-7天）。 |
| **申诉流程** | 公示页“我要申诉”入口，需在公示期内提交理由、证据与联系方式；后台按“受理→调查→处理”SLA（1+3-5+1工作日）执行并记录全链路。 |
| **处理结果** | 维持原判、重新评审、取消中标、追加奖励等多种结果，并实时通知相关角色并更新公示页。 |
| **协议签订与支付** | 公示期结束且无异议后自动进入线上协议签署，学生提交收款信息，学校审核后由企业/平台打款并由学生确认收款，自动生成到账凭证。 |
| **归档与复盘** | 系统打包方案、执行记录、评审数据、支付凭证、沟通纪要等形成归档包，供后续统计、复盘与知识沉淀；同时生成流程KPI（方案审核时长、逾期率等）。 |

## 四、非功能性需求

| 需求类型 | 具体要求 |
|---------|---------|
| **性能需求** | • 系统响应时间：页面加载 < 2秒<br>• 并发用户支持：至少1000人同时在线<br>• 文件上传：支持单文件最大500MB<br>• 文件下载：多线程下载，断点续传 |
| **安全需求** | • 用户密码加密存储（MD5+盐值）<br>• 数据传输HTTPS加密<br>• SQL注入防护<br>• XSS攻击防护<br>• 文件上传安全检查<br>• 敏感操作日志记录<br>• 权限管理（RBAC） |
| **可用性需求** | • 界面友好，符合用户使用习惯<br>• 响应式设计，支持PC、平板、手机<br>• 操作引导和帮助文档<br>• 错误提示清晰明确<br>• 支持主流浏览器（Chrome、Firefox、Edge、Safari） |
| **可靠性需求** | • 系统可用性 ≥ 99.5%<br>• 数据备份：每日自动备份<br>• 容灾恢复：支持快速恢复<br>• 异常处理机制完善 |
| **可扩展性需求** | • 模块化设计<br>• 支持功能模块动态添加<br>• 支持接口扩展<br>• 数据库支持水平扩展 |
| **兼容性需求** | • 支持Windows、macOS、Linux<br>• 支持主流浏览器最新版本及前两个版本<br>• 支持移动端浏览器 |

## 五、系统流程设计

### 5.1 项目发布流程
```
企业提交项目 → 学校审核 → 审核通过/驳回
                ↓
         项目发布成功 → 学生浏览/揭榜
```

### 5.2 揭榜流程
```
学生查看项目 → 个人/组队 → 提交揭榜申请 
                              ↓
              企业审核（可选）/自动通过
                              ↓
                        揭榜成功 → 开始项目
```

### 5.3 成果评审流程
```
学生提交成果 → 学校初审 → 初审通过
                              ↓
               企业+专家评审 → 评分排名
                              ↓
                    确定中标 → 公示期
                              ↓
                    公示完成 → 协议签订
```

### 5.4 支付流程
```
协议签订完成 → 学生提交收款信息 → 学校审核
                                      ↓
                  平台/企业发放奖金 → 学生确认收款
                                      ↓
                              项目完成归档
```
