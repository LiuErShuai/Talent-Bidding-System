# 背景与目标

- **背景**：私信模块已具备会话列表与聊天窗口，但会话区域/输入框出现滚动条影响视觉一致性，且输入框高度偏大。
- **目标**：隐藏会话列表与聊天内容区滚动条（保持可滚动），同时减小输入框高度并隐藏输入框内部滚动条（仍可滚动编辑）。

## 目录

1. 需求与验收标准
2. 实施方案
3. Todolist（时间戳原则）
4. 风险与回滚

## 1. 需求与验收标准

- 会话列表与聊天记录区域滚动条隐藏，但仍可鼠标滚轮/触控板滚动。
- 私信输入框高度减小为固定高度，内容超出仍可在输入框内部滚动编辑。
- 输入框内部滚动条隐藏（仍可滚动）。

## 2. 实施方案

- 对 `.message-list`、`.conversation-list`、`.chat-scroller`、`.contact-list` 添加隐藏滚动条样式：
  - `scrollbar-width: none;`（Firefox）
  - `-ms-overflow-style: none;`（旧 Edge/IE）
  - `::-webkit-scrollbar { display: none; }`（Chromium/WebKit）
- 私信输入框：
  - `rows` 调整为更小；
  - `.el-textarea__inner` 固定高度并 `overflow-y: auto`；
  - 同时对 `.el-textarea__inner` 的滚动条做隐藏处理。
