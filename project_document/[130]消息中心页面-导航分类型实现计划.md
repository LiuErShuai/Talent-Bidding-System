# 背景与目标

- **背景**：当前 `src/views/message/MessageCenter/index.vue` 为空白页面，需要实现“消息中心/信息中心”页面，并参考 CSDN 消息中心的交互形态。
- **目标**：提供一个可用的消息中心页面，包含消息类型导航栏（我的消息、收藏关注、评论和回复），支持左侧列表 + 右侧详情展示，便于后续接入真实 API。

2. 技术方案与取舍

## 1. 需求与验收标准

- 页面标题为“消息中心”，并提供顶部消息类型导航栏。
- 消息类型至少包含：
  - 我的消息
  - 收藏关注
  - 评论
  - 回复
- 切换类型后左侧列表随之变化；点击列表项后右侧显示详情；未选择时右侧展示空状态提示。
- 视觉结构参考 CSDN：顶部标题 + Tabs，主体为左右两栏（左列表右详情），整体风格与当前站点浅色背景一致。

## 2. 技术方案与取舍

### 方案 A：Element Plus Tabs + 自定义左右布局（推荐）

- 使用 `el-tabs` 实现类型导航，`el-empty/el-avatar/el-badge/el-input` 作为基础交互组件。
- 优点：开发快、风格统一、交互一致性好；后续接入 API 方便。
- 缺点：需要适配 Element Plus 的默认样式（通过 scoped CSS 进行覆盖）。

### 方案 B：纯 HTML/CSS 实现 Tabs + 列表/详情

- 优点：样式可完全自定义，接近截图细节更容易。
- 缺点：交互细节（激活态、徽章、空状态）需要重复造轮子。
