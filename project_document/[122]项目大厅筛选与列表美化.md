# [122] 项目大厅筛选与列表美化

## 背景与目标
当前 `项目大厅`（`src/views/project/Hall/index.vue`）的筛选项较少，且项目列表卡片风格与首页 `任务列表`（`src/views/home/index.vue`）不一致。为了提升可用性与视觉一致性，需要补充“项目领域”筛选项，并引入“更多”入口，同时对项目列表卡片做对标美化。

**目标：**
1. “项目领域”新增若干常用领域，并在末尾增加“更多”筛选入口（展开/收起更多领域）。
2. 项目列表卡片布局与标签（状态标签/按钮）对齐首页任务列表风格，提升信息密度与可读性。
3. 不改变现有筛选/排序逻辑的业务含义，仅做 UI 与交互增强。

## 1. 范围与验收标准
**范围：**
- `src/views/project/Hall/index.vue`：筛选区（项目领域）、项目列表卡片样式与结构。
- 必要时同步少量全局样式（如保持一致性），优先放在页面 scoped 样式中。

**验收标准：**
- 项目领域行新增多个领域，并提供“更多”入口，可展开选择更多领域。
- 选择“更多”中的领域后，筛选生效且 UI 显示为已选中状态。
- 项目列表卡片：状态标签、奖励金额、详情/揭榜按钮等布局更清晰，整体风格接近首页任务列表。
- 页面可正常渲染、交互无报错。

## 2. 实施方案与取舍
**方案 A：直接把所有领域全部平铺**
- 优点：实现最简单。
- 缺点：领域多时占用空间大，移动端换行过多影响阅读。

**方案 B：固定展示部分领域 + “更多”弹出选择（采用）**
- 优点：保持常用选项可见，同时兼顾扩展性；对齐首页已有“更多领域”模式。
- 风险：需要处理“选中项在更多里”的高亮与关闭逻辑（可控）。
