## 一、背景与需求

### 1.1 项目背景

项目将来需要部署到校园内网环境，现有的首页（`/home`）将作为外网的门户页面，而除此之外的所有功能页面都需要经过内网登录后才能查看。因此需要新增一个校内首页，作为内网环境的入口页面。

### 1.2 核心需求

1. **功能定位**：校内首页作为内网环境的主入口，用户登录后的默认落地页
2. **内容展示**：展示内容与当前首页类似，但更简约紧凑
3. **页面布局**：一屏展示完整内容，无需滚动，简约高效
4. **访问控制**：必须登录后才能访问，未登录自动跳转登录

### 1.3 设计原则

- **简约紧凑**：去除冗余信息，一屏展示核心内容
- **高效导航**：快速访问常用功能，减少操作步骤
- **角色适配**：根据用户角色展示个性化内容
- **响应式设计**：适配不同屏幕尺寸

### 2.3 去除的内容

相比外网首页，校内首页**不展示**以下内容：

1. ❌ **欢迎区域**（Hero Section）- 完全去除
2. ❌ **统计数据**（创客任务总数、创客人数、悬金池总额等）
3. ❌ **流程展示区域**（任务应征、成果生成、成果提交、成果发表）
4. ❌ **筛选器区域**（任务领域、任务状态筛选）
5. ❌ **统计图表**（发布任务数、任务总金额、饼图等）
6. ❌ **合作企业模块**
7. ❌ **页脚（Footer）**
8. ❌ **快速入口区域**（导航栏已包含这些功能）
9. ❌ **所有图标元素**（包括 emoji 和图标字体）

### 2.4 样式设计要求

**背景设计**：
- 使用与外网首页相同的背景图：`@/assets/images/backgrounds/home_bg.jpg`
- 背景处理方式：
  - 透明度：0.15（淡化处理）
  - 渐变遮罩：从上到下渐变消失
  - 跟随页面滚动

**色彩规范**：
- 主色调：#2563eb（蓝色）
- 背景色：#f5f7fb（浅灰）
- 卡片背景：#ffffff
- 文字颜色：#1f274b（深蓝灰）
- 次要文字：#7b859f（灰色）
- 避免使用过多彩色元素，保持简洁专业

**布局规范**：
- 最大宽度：1200px（居中显示）
- 边距：24px
- 卡片圆角：8px
- 卡片阴影：0 2px 8px rgba(0, 0, 0, 0.05)
- 间距：24px

### 2.5 页面尺寸规划

**目标**：尽量在一屏内展示（1920x1080），允许少量滚动

- **导航栏**：60px
- **主内容区**：根据实际内容自适应
  - 热门项目列表：左侧 2/3 宽度
  - 侧边栏：右侧 1/3 宽度
- **间距**：24px

**说明**：由于去除了欢迎区域和快速入口，页面高度会自然减少，具体高度在实现后根据实际效果微调。

## 三、路由与导航设计

### 3.1 路由配置

**路由路径**：`/campus-home`

**路由元信息**：
```javascript
{
  path: '/campus-home',
  name: 'CampusHome',
  component: () => import('@/views/home/CampusHome.vue'),
  meta: {
    title: '校内首页',
    requiresAuth: true,  // 必须登录
    layout: 'default',   // 使用默认布局（含导航栏）
    transition: 'fade'
  }
}
```

### 3.2 导航栏设计说明

**重要说明**：
- **校内首页（`/campus-home`）及所有内网功能页面**：使用现有的通用导航栏，**完全不需要修改**
- **外网首页（`/home`）**：需要修改导航栏，改为展示型导航

#### 3.2.1 校内首页导航栏（保持现状）

**说明**：校内首页及所有内网功能页面的导航栏保持现有设计不变。

**现有导航栏特点**：
- Logo + 平台名称（创客平台）
- 功能菜单：首页、项目大厅、我的项目
- 右侧显示用户头像 + 下拉菜单（登录后）或登录/注册按钮（未登录）

**无需任何修改**，继续使用现有的导航栏组件。

#### 3.2.2 外网首页导航栏（需要修改）

**说明**：外网首页（`/home`）的导航栏需要修改为展示型导航，去除功能性菜单。

**修改前（现有的功能型导航）**：
```
[Logo] 创客平台  [首页] [项目大厅] [我的项目]  |  [登录] [注册]
```

**修改后（展示型导航）**：
```
[Logo] 创客平台  [首页] [合作企业] [关于我们] [帮助中心]  |  [校内登录]
```

**修改内容**：
- ❌ 去除功能性导航：项目大厅、我的项目
- ✅ 添加展示性导航：合作企业、关于我们、帮助中心
- ✅ 保留"首页"导航
- ✅ 将"登录"改为"校内登录"
- ❌ 去除"注册"按钮

**导航项说明**：
- **首页**：跳转到 `/home`（外网首页）
- **合作企业**：跳转到 `/partners` 或锚点到合作企业模块
- **关于我们**：跳转到 `/about`
- **帮助中心**：跳转到 `/help`
- **校内登录**：调用现有的 `openAuthDialog('login')` 方法，复用 `AuthDialog` 组件弹出登录弹窗，登录成功后跳转到 `/campus-home`

### 3.3 导航逻辑

**登录后默认跳转**：
- 用户登录成功后，默认跳转到 `/campus-home`
- 如果有 `redirect` 参数，则跳转到指定页面

**首页切换**：
- 外网首页：`/home`（无需登录，展示型页面）
- 校内首页：`/campus-home`（需要登录，功能型页面）

**面包屑导航**：
- 校内首页不显示面包屑
- 其他页面显示：校内首页 > 当前页面

## 四、权限与拦截方案

### 4.1 路由守卫拦截

**拦截逻辑**（在 `src/router/guards.js` 中实现）：

```javascript
// 校内首页必须登录
if (to.path === '/campus-home' && !authStore.isLoggedIn) {
  // 触发登录弹窗
  window.dispatchEvent(
    new CustomEvent('open-auth-dialog', {
      detail: { mode: 'login', redirect: '/campus-home' }
    })
  )
  // 跳转到外网首页
  next('/home')
  return
}
```

### 4.2 登录后跳转逻辑

**修改登录成功后的跳转逻辑**（在 `src/store/modules/auth.js` 中）：

```javascript
// 登录成功后
async login(credentials) {
  // ... 登录逻辑

  // 登录成功后跳转
  const redirect = sessionStorage.getItem('loginRedirect') || '/campus-home'
  sessionStorage.removeItem('loginRedirect')
  router.push(redirect)
}
```

### 4.3 外网与内网的区分

**外网页面**（无需登录）：
- `/home` - 外网首页
- `/about` - 关于我们
- `/contact` - 联系我们
- `/help` - 帮助中心
- `/privacy` - 隐私政策

**内网页面**（需要登录）：
- `/campus-home` - 校内首页
- `/projects` - 项目大厅
- `/my-projects` - 我的项目
- `/smart-match` - 智能匹配
- `/messages` - 消息中心
- 其他所有功能页面

### 4.4 导航栏显示逻辑

**外网首页（`/home`）导航栏**：
- 显示：首页、合作企业、关于我们、帮助中心
- 右侧：[校内登录] 按钮
- 点击"校内登录"调用现有的 `openAuthDialog('login')` 方法触发登录弹窗（**复用现有 AuthDialog 组件，不需要重新开发**）
- **需要修改**：将现有的功能型导航改为展示型导航

**校内首页（`/campus-home`）及所有内网功能页面导航栏**：
- 功能菜单：首页、项目大厅、我的项目
- 右侧：**仅显示**用户头像 + 下拉菜单（已登录状态）
- ❌ **不需要**登录/注册按钮（进入内网页面必须已登录）
- 可复用现有 AppHeader 组件的已登录状态逻辑

## 八、风险与应对

### 8.1 潜在风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 后端 API 未就绪 | 无法获取数据 | 中 | 使用 Mock 数据开发，后续对接 |
| 卡片高度压缩困难 | 信息展示不完整 | 低 | 采用双行布局，保留核心信息 |
| 背景样式不一致 | 视觉效果差异 | 低 | 复用外网首页的背景样式代码 |
| 响应式适配困难 | 移动端体验差 | 低 | 优先保证桌面端，移动端简化展示 |

### 8.2 降级方案

**如果卡片压缩效果不理想**：
- 方案1：采用双行布局，保留更多信息
- 方案2：减少显示的项目数量（从 3-4 个减少到 2-3 个）
- 方案3：使用折叠展开方式显示详细信息
