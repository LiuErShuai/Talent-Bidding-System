# [120]学生个人中心页面开发设计文档

## 一、背景与目标

### 1.1 背景
当前个人中心页面（`src/views/user/Profile/index.vue`）显示的内容实际上是帮助中心页面，需要重新设计并实现一个真正的个人中心页面。本次开发先针对学生用户，后续可扩展至企业、教师等角色。

### 1.2 目标
1. 设计并实现学生用户的个人中心页面，包括：
   - **个人资料**：基本信息展示与编辑（学号、姓名、学院、专业、年级、联系方式、头像等）
   - **详细资料**：个人技术简介、项目经历等扩展信息
2. 将帮助中心页面从个人中心路由移除，添加到头像下拉菜单中
3. 确保页面UI美观、交互流畅，符合项目整体设计风格

## 三、需求分析

### 3.1 功能需求

#### 3.1.1 个人资料（基本信息）
- **展示字段**：
  - 头像（可上传/编辑）
  - 用户名/姓名
  - 学号（只读）
  - 学院
  - 专业
  - 年级
  - 手机号
  - 邮箱
  - 注册时间（只读）
  - 上次登录时间（只读）
- **功能**：
  - 支持编辑模式，可修改基本信息
  - 头像上传（支持图片预览、裁剪）
  - 表单验证（手机号、邮箱格式）
  - 保存/取消操作

#### 3.1.2 详细资料（扩展信息）
- **技术简介**：
  - 个人技术简介（多行文本，支持Markdown或富文本）
  - 技能标签管理（已有功能，需保留）
- **项目经历**：
  - 展示参与的项目列表
  - 项目名称、项目状态、参与角色、完成时间
  - 可跳转到项目详情页
- **其他信息**（可选）：
  - 获奖经历
  - 竞赛经历
  - 实习经历

### 3.2 非功能需求
- 响应式设计，支持移动端
- 页面加载性能优化
- 数据持久化（与后端API对接）
- 错误处理和用户提示

#### 5.1.2 个人资料卡片设计
- **展示模式**：
  - 信息以卡片形式展示，每行一个字段
  - 只读字段（学号、注册时间等）使用灰色文字
  - 可编辑字段显示编辑按钮
- **编辑模式**：
  - 点击"编辑信息"按钮进入编辑模式
  - 表单字段使用Element Plus的`el-input`、`el-select`等组件
  - 底部显示"保存"和"取消"按钮

#### 5.1.3 详细资料卡片设计
- **技术简介区域**：
  - 使用`el-input`的`type="textarea"`，支持多行输入
  - 显示字符计数（如：500/1000字）
  - 支持Markdown预览（可选）
- **项目经历区域**：
  - 使用列表或卡片形式展示
  - 每个项目显示：项目名称、状态标签、参与角色、时间范围
  - 点击项目可跳转到项目详情页
  - 空状态提示："暂无项目经历"

### 5.2 交互流程

#### 5.2.1 编辑个人信息流程
```
1. 用户点击"编辑信息"按钮
2. 页面切换到编辑模式，表单字段变为可编辑
3. 用户修改信息
4. 点击"保存"按钮
   - 前端表单验证
   - 调用API更新用户信息
   - 显示成功提示
   - 退出编辑模式
5. 点击"取消"按钮
   - 恢复原始数据
   - 退出编辑模式
```

#### 5.2.2 上传头像流程
```
1. 用户点击头像区域（编辑模式下）
2. 触发文件选择对话框
3. 用户选择图片文件
4. 前端验证（文件类型、大小）
5. 显示图片预览
6. 调用API上传头像
7. 更新头像显示
```
