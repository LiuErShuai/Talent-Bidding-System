# [127]消息中心高度缩小与隐藏滚动条

> 背景与目标：用户希望消息中心页面整体更“矮”一些（主区域高度缩小），同时 `list-area` 与 `chat-scroller` 仍可滚动但不显示滚动条，保持界面干净。

- 4. 执行记录（时间戳）
- 5. 审查与结论

## 1. 需求与验收标准
- `list-area` 与 `chat-scroller` 依然支持滚动（鼠标滚轮/触控板/触摸）。
- 两个区域滚动条隐藏（不出现可见滚动条），但不影响内容可达性。
- 主体区域高度相较之前缩小，页面视觉留白更明显。

## 2. 方案说明
- 高度缩小：对主体容器增加 `max-height`（基于视口高度计算），让左右面板的可视高度随之降低。
- 隐藏滚动条：使用跨浏览器样式（Firefox 的 `scrollbar-width:none` + WebKit 的 `::-webkit-scrollbar`）隐藏滚动条但保留滚动能力。
