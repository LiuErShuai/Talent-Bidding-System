## 研究与调查摘要
- **需求依据**：参考 `[110]项目里程碑页面优化开发方案.md`，确认七阶段结构、卡片内容与详情弹窗要求。
- **现状分析**：`src/views/project/Tracker/index.vue` 仍为单纯进度条 + 时间轴，数据结构简单，缺少所需 UI 及逻辑。

## 执行策略
1. 以静态示例数据驱动 UI，优先完成布局与交互骨架，后续可替换为真实 API。
2. 将工具函数（如描述截断、剩余天数计算）内联至页面，保持组件自洽。
3. 采用 Element Plus 卡片、步骤条、弹窗等组件组合实现，重点关注中文注释与响应式细节。

## 风险与依赖
- UI 细节较多，需对照方案逐项校验，避免遗漏。
- 真实数据 API 尚未接入，目前以 mock 数据为准，后续需留扩展点。
- 弹窗内容丰富，需关注滚动与性能，保证在低分辨率下展示得当。

## 验收要点
- 七阶段进度与数据结构保持一致，卡片与详情信息完整。
- 当前里程碑卡片始终可用，状态切换逻辑正确。
- 截断、剩余天数等辅助信息符合方案描述，样式响应式良好。
