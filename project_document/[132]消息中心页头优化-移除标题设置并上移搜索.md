# 背景与目标

- **背景**：当前消息中心页面顶部存在“消息中心”标题与“消息设置”按钮，搜索框位于左侧列表顶部。为贴近参考样式与更紧凑的页头布局，需要调整信息层级。
- **目标**：移除标题与“消息设置”，将搜索条移动到顶部导航栏区域（分类 Tabs 同一行右侧），并保持原有搜索过滤逻辑不变。

## 1. 需求与验收标准

- 去除导航栏上方的“消息中心”标题展示。
- 去除“消息设置”按钮。
- 将搜索条从左侧列表顶部移动到上方导航栏区域（与 Tabs 同一行，靠右展示）。
- 原搜索功能仍然生效：输入关键字可过滤当前分类下的消息列表。
- 响应式下（窄屏）搜索条可自动换行，不影响 Tabs 点击。

## 2. 技术方案

- 采用现有 Element Plus `el-tabs`，在 Tabs 外层容器中增加 `header-tools`，使用 Flex 布局把 Tabs 与搜索框放到一行。
- 删除左侧 `left-toolbar` 区块，避免重复搜索入口。
- 使用 `scoped` 样式对布局进行轻量调整，不引入新的组件与路由变更。

## 4. 风险与回滚

- 风险：Tabs 与搜索框同一行在不同宽度下可能挤压布局。
  - 处理：使用 `flex-wrap`，并为搜索框设置合理的最小/最大宽度。
- 回滚：本次仅涉及单页面与文档调整，可通过 `git revert` 回滚单次提交。
